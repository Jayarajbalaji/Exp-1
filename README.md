# Exp-1 ATM & BANKING SYSTEM

# AIM

To study and understand the functional behavior of an Automated Teller Machine (ATM) system and Banking System by preparing the Software Requirement Specification (SRS) and designing various UML diagrams.

# SRS (Procedure)
# 1. Introduction

The Automated Teller Machine (ATM) system allows bank customers to perform basic transactions like cash withdrawal, balance inquiry, mini statements, fund transfer, etc., without human intervention.

# 2. Purpose

The purpose of this project is to automate the banking operations performed through an ATM and describe the system functionalities using UML diagrams.

# 3. Scope

The system includes:

Customer authentication

PIN validation

Cash withdrawal

Balance inquiry

Transaction processing

Communication between ATM and Bank Server

Transaction logs

# 4. Functional Requirements

FR1: The system shall authenticate the customer using ATM card and PIN.

FR2: The system shall allow withdrawal of valid amounts from the customer account.

FR3: The system shall display the available balance.

FR4: The ATM must communicate with the bank server for transaction approval.

FR5: The system shall print receipts.

# 5. Non-Functional Requirements

Performance: Response time < 5 seconds

Reliability: 24/7 availability

Security: Encrypted communication

Usability: Simple and user-friendly interface
